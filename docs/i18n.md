# Internationalization (i18n) guide

Kotoba's UI strings, error messages, and docs are translatable. This guide explains how to add a new language.

## Languages we want first

Beyond English (default) and Japanese, we'd love early support for:

- Spanish (es)
- Portuguese (pt-BR)
- Vietnamese (vi)
- Indonesian (id)
- French (fr)
- German (de)
- Korean (ko)
- Mandarin (zh-CN, zh-TW)
- Thai (th)

If your language isn't here, that doesn't mean we don't want it. Open an issue with `i18n: <language>`.

## How it works

UI strings live in [Project Fluent](https://projectfluent.org/) `.ftl` files at:

```
crates/kotoba-cli/locales/
├── en/
│   └── kotoba.ftl
├── ja/
│   └── kotoba.ftl
└── <your-lang>/
    └── kotoba.ftl
```

Fluent supports plurals, gendered forms, and message-level fallbacks — important for languages where a literal string-table doesn't suffice.

## Adding a new locale

1. Open an issue `i18n: <language>`. A maintainer assigns it to you and writes a brief locale-style guide (formality, honorifics, technical vocabulary).
2. Copy `crates/kotoba-cli/locales/en/kotoba.ftl` to `crates/kotoba-cli/locales/<your-lang>/kotoba.ftl`.
3. Translate the right-hand side of each entry. Leave keys untouched.
4. Add a line to `LOCALES` in `crates/kotoba-cli/src/i18n.rs` (a maintainer can do this for you).
5. Open a PR.

## Translating docs

For docs, we use a simpler convention: copy `README.md` to `README.<lang>.md`. Same for `CONTRIBUTING`, `CODE_OF_CONDUCT`, etc. Link the translated docs in the original via the language switcher near the top.

We do not require docs translations to be 1:1 with English. Cultural adaptation is welcome.

## Style guide

- **Match the user's expectation in your language.** Don't translate "deck" literally if your language uses something else for flashcard collections.
- **Keep technical terms when they're well-known.** "JLPT", "FSRS", "MCP" generally stay as-is.
- **Test in your shell.** `LC_ALL=<your-locale> kotoba review` should look right.
- **Watch line widths.** Some languages take more horizontal space; check that TUI screens don't overflow.

## Crediting translators

Add yourself to the locale file header:

```ftl
# Kotoba — <Language> translation
# Translators:
#   - your-handle (year)
```

You'll also be added to the `README.<lang>.md` translators block.
